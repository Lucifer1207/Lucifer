( UNIT 1 )-- * ADRESSING MODES *

" The Different ways of specifying
the location of an operand in an instruction
are called as ADDRESSING MODES."



* TYPES OF ADDRESSING MODES

There are mainly 10 types of Addressing Modes :-

  • Implied Addressing Mode.
  • Immediate Addressing Mode.
  • Direct Addressing Mode.
  • Indirect Addressing Mode.
  • Register-direct Addressing Mode.
  • Register-Indirect Addressing Mode.
  • Stack Addressing Mode.
  • Relative Addressing Mode.
  • Index Addressing Mode.
  • Base Register Addressing Mode.



Now , Let's learn them one by one.....

1): Implied Addressing Mode

In this addressing mode ,
the defination of the instruction itself specify the operands implicitly.

e.g. Complement Accumulator is an Implied Mode.




2): Immediate Addressing Mode

i) : In this addressing mode ,
The Operand is specified in the instruction explicitly.

ii) : Instead of address field , an operand field is present that contains the operand.

e.g. ADD 10 is an IMMEDIATE Mode.




3): Direct Addressing Mode

In this addressing mode ,
i) : The Adress field of the instruction contains the EFFECTIVE ADDRESS of operand.

ii) : Only ONE reference to the memory is required to fetch the operand.

e.g. ADD X is an DIRECT Mode.




4): Indirect Addressing Mode

In this addressing mode ,
i) : The Address field of the instruction contains the address of Memory location that contains the EFFECTIVE ADDRESS of operand.

ii) : TWO references of the memory are required to fetch the operand.

e.g. LOAD R1 is an INDIRECT Mode.







5): Register-direct Addressing Mode

In this addressing mode ,
i) : The Adress field of the instruction refers to CPU register that contains the operand.

ii) :NO refernce to the Memory is required.

e.g. ADD R is an REGISTER-DIRECT Mode.







6): Register-Indirect Addressing Mode

In this addressing mode ,
i) : THE ADDRESS field of the instruction refers to the CPU register that contains the EFFECTIVE ADDRESS of operand.

ii) : ONE references of the memory are required to fetch the operand.

e.g. LOAD R1 is an REGISTER-INDIRECT Mode.







7): Stack Addressing Mode

In this addressing mode ,
"The operand is contained at the TOP of the Stack."

e.g. ADD is an STACK Mode.







8): Relative Addressing Mode

In this addressing mode ,
i) : The EFFECTIVE ADDRESS of the operand is obtained by adding the content of PROGRAM COUNTER to the address part of the instruction.

ii) : The Adress part of instruction can be either (+ve) or (-ve) represented in 2's Complement.

e.g. ADD R1,4000 is an RELATIVE Mode.







9): Index Addressing Mode

In this addressing mode ,
i) : The EFFECTIVE ADDRESS of the operand is obtained by adding the content of INDEX REGISTER to the address part of the instruction.

ii) : tHE dISTEANCE BETWEEN the base address and the address of operand is the INDEX VALUE stored in index register.

e.g. Arrays is an INDex Mode.







10): Base register Addressing Mode

In this addressing mode ,
i) : The EFFECTIVE ADDRESS of the operand is obtained by adding the content of BASE REGISTER to the address part of the instruction.

ii) : used for PROGRAM RELOCATION
